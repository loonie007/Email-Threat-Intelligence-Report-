Email Threat Intelligence Project

A Cybersecurity Case Study by Samuel Akinleye

Project Introduction

This project demonstrates a professional email phishing investigation using a structured cybersecurity workflow. It replicates a real-world Security Operations Centre (SOC) analysis process, where a suspicious email impersonating Microsoft was reported by an employee.

Through manual inspection, open-source intelligence tools, and command-line utilities, the project identifies spoofed domains, malicious URLs, and unauthorized email authentication failures.

Objective

The objective of this project is to analyze and interpret phishing activity by:

  • Performing in-depth header and body analysis of suspicious emails.
  • Utilizing open threat intelligence platforms to validate IOCs.
  • Recommending measures to enhance corporate email security posture.

Key Findings

  • Sender domain access-accsecurity.com identified as spoofed.
  • Failed SPF, DKIM, and DMARC authentication checks.
  • Malicious URLs sign.in and thebandalisty.com confirmed phishing indicators.
  • Origin IP 89.144.44.41 traced to a suspicious hosting provider.
